Error Types

ChromeDevToolsLite defines several error types for different failure scenarios:

ChromeDevToolsLite.ElementNotFoundError — Type
ElementNotFoundError

Thrown when an element cannot be found in the DOM using the specified selector.

source
ChromeDevToolsLite.NavigationError — Type
NavigationError

Thrown when a page navigation fails or times out.

source
ChromeDevToolsLite.EvaluationError — Type
EvaluationError

Thrown when JavaScript evaluation in the browser fails.

source
ChromeDevToolsLite.TimeoutError — Type
TimeoutError

Thrown when an operation exceeds its time limit.

source
ChromeDevToolsLite.ConnectionError — Type
ConnectionError

Thrown when there are issues with the WebSocket connection.

source

Error Handling Examples

# Handle element not found
try
    element = ElementHandle(client, "#non-existent")
catch e
    if e isa ElementNotFoundError
        @warn "Element not found" selector="#non-existent"
    end
end

# Handle navigation errors
try
    goto(client, "invalid-url")
catch e
    if e isa NavigationError
        @error "Navigation failed" url="invalid-url" reason=e.msg
    end
end

# Handle evaluation errors
try
    result = evaluate(client, "invalid javascript")
catch e
    if e isa EvaluationError
        @error "JavaScript evaluation failed" reason=e.msg
    end
end

# Handle timeouts
try
    element = ElementHandle(client, "#slow-loading", timeout=1.0)
catch e
    if e isa TimeoutError
        @warn "Operation timed out" operation="element selection"
    end
end

# Handle connection errors
try
    client = connect_browser()
catch e
    if e isa ConnectionError
        @error "Failed to connect to browser" reason=e.msg
    end
end
